爆料革命网络宣传单制作手册

为什么要这样制做宣传单：网络宣传单要求是富文本(有图有文)，所以可选方案有：word / pdf / picture / markdown / html。其中 word / pdf 都有专有病毒，无法保证安全性，所以必须放弃。picture 已经很难加载病毒(历史上曾出现过图片病毒，但现在已经几乎没有了)，但图片中所有文字修改难，不可复制，传播过程中很容易被网监识别散列值，所以，很难在墙内传播，战友经常在图片上画点线传播，这样会破坏图片中的有效元素，所以不完全适合爆料传播。markdown 是一种类html语言，编写比html简单，但，现阶段还未实现全操作系统支持，也就是未流行，同时，也无法在同一文件内加载图片，所以，不完全适合。
而 html 做为成熟技术，熟悉此技术的人力资源足够，已经实现全操作系统支持，图文在同一文档加载，修改容易，本身是纯文本无法加载病毒，所有文本可复制，传播中对html标签较小的修改不会影响内容展现。因此是传播爆料的最好选择方式。同时，放到任何网站时，都可以与网站融为一体，大部分网络教程也很容易转成这种格式。这种方式做出来的宣传单随时可以生成图片传播。

一、注意事项：

1. 不加脚本：现在脚本已经很难实现对操作系统的攻击，但链接还是可以通过跳转钓鱼的。所以在宣传单内不加任何脚本类元素。
2. 写入图片：所有图片以 base64 文本形式写入html，这样整个宣传单会以单一文件形式存在，传播时不会出现图片无法加载。
3. 少加链接：链接跳转有钓鱼嫌疑，因此尽量要少加，如果一定要加，必须是可信链接。我会监督群里此类宣传单中的链接。
4. 写入样式：样式表不用link，直接写入html文件内，保证美观，传播中又不丢失文件。要尽量保证样式表内容简洁、有效。

二、实现方式：

html的编写就不多讲了，只提一下，各农场可以根据自己的特点和实力做一套模板，并对所有以此模板发出的文档进行编号管理，防止匪类借此宣传。

1. 编写规则：

所有文档编码一定是utf-8编码，这样适用全世界所有语言操作系统浏览，不会出现乱码，也就是 <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />

2. 生成编码：

要把文件直接加入到html中，一定要将文件生成base64纯文本，然后将文本按一定规则写到html中，以下为图片生成base64文本命令： python -m base64 aa.png > aa.txt 将图片生成base64文本，此方法在 mac / win / linux 上都适用，只不过 win 需要安装python。也可以到网上找在线转码工具。因视频转码网站不好找，下边给大家找好了。
如果要将其它类型文件生成base64，将"aa.png"换成其它文件就可以。"aa.txt"打开就可以看到生成的base64编码。这里会按一定长度换行，这个换行保留与否都可以，因为html中是不认换行符的。
注：
  视频转base64 - https://base64.guru/converter/encode/video
  后边的"[base64]"就是要替换成这里生成的base64编码文本。
  某些网上转码工具会直接生成html编码，直接替换就可以。

3. 图片加载：

如果将base64编码的图片文本加入html，直接写入以下代码。 <img src="data:image/png;base64,[base64]"> 这里的"png"是图片格式，要根据你的图片后缀修改，如：jpg / gif。此种写法，在标签内加其它属性依然适用，如：width，后边几个也一样，不再提了。

4. 下载链接：

<a href="data:ttf/plain;base64,[base64]"> 这里的"ttf"是字体文件，要根据你的文件后缀修改，我这里是下载字体。这里在下载时可通过另存或点击下载，但都要修改文件名和后缀，请在链接后注释。

5. 背景图片：

<div style="background: url(data:image/gif;base64,[base64]); padding: 20px 0 0 0;"> css另一种写法 <style>
  .aa {background: url(data:image/gif;base64,[base64]);}
</style>
<div class="aa"></div> 这里也可以添加背景图片的其它CSS属性。

6. 视频加载：

<video controls="controls" autobuffer="autobuffer" autoplay="autoplay">
  <source src="data:video/mp4;base64,[base64]">
</video> 这里的"mp4"根据你自己的文件后缀替换。只要html支持的就可以。

7. 音频加载：

<audio controls="controls" autobuffer="autobuffer" autoplay="autoplay">
  <source src="data:audio/wav;base64,[base64]" />
</audio> 这里的"wav"根据你自己的文件后缀替换。只要html支持的就可以。

8. 字体加载：

<style type="text/css">
  @font-face {
    font-family: "myfont";
    src: url(data:font/truetype;charset=utf-8;base64,[base64]);
    }
  body { font-family: "myfont" }
</style> 这里一定要记住在前边加标签定义utf-8编码，要不然中文全是乱码。

9. 加载脚本：

<script src="data:text/javascript;base64,[base64]">
</script>

10. 加载框架：

框架内网页：
<iframe src="data:text/html;base64,[base64]" height="100%" width="100%">
</iframe> 框架内文档：
<iframe src="data:application/pdf;base64,[base64]" height="100%" width="100%">
</iframe>

10. 收藏图标：

ICO收藏夹图标某些浏览器不认，所以没意义。这里就不讲了。

11. 换行问题：

html纯文本中换行符在不同的操作系统里不一样，在linux体系中是"\n"，而在win体系中是"\n\r"，所以，当战友看源码时最好用UE之类的软件打开，要不然整个源码可能会成为一行，显得很乱。也可以用浏览器打开，然后查看源码，将源码复制到文本中。

三、其它实现：

1. 隐藏新闻：

点击新闻标题，下拉显示内容
<script>
  function showCo(data) {
  myElm = data.nextElementSibling;
  myElm.style.display=="block"?myElm.style.display="none":myElm.style.display="block";
  }
</script>

<div>
  <div onclick="showCo(this)">新闻标题</div>
  <div style="display: none;">新闻内容</div>
</div> 将以上代码分别加到html的脚本区和body区。div标签可以选择其它标签实现。标题和内容标签可以不相同，但要保证标题和内容在相同的父标签中，标题与内容间隔出现。

2. ps图片如何生成html：

首先，在ps中打开一张要处理的图片。
然后，在左侧的工具栏中点击切片工具。
接着，用切片工具把图片分成4份。
之后，再依次点击左上角的文件——存储为web所用格式。在弹出的窗口中点击存储的按钮。
最后，弹出的保存窗口中把格式选择为HTML图像，再点击保存按钮即可。
